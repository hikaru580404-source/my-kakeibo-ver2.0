# マイ家計簿 — README

## プロジェクト概要

スマートフォンファースト設計の個人・法人兼用家計簿アプリ。  
現金残高と銀行口座残高を分けて管理し、日次入力・月次サマリー・予算管理・月末棚卸しを一貫して運用できる。

---

## ページ構成と機能

| ページ | URL | 概要 |
|---|---|---|
| ダッシュボード | `index.html` | KPI・グラフ・明細リスト・棚卸しバナー |
| 入力フォーム | `form.html` | 収支3ステップ入力（STEP1→確認→完了） |
| 月次サマリー | `summary.html` | 振り返り・比較グラフ・棚卸し差異履歴 |
| 予算管理 | `budget.html` | カテゴリ別予算入力・前月実績コピー |

---

## 完成済み機能

### ダッシュボード（index.html）
- KPI 4枚：今月収入 / 今月支出 / 現在残高 / 月末着地予測
- 銀行口座 / 現金残高の内訳表示
- 月次収支推移バーチャート（過去6ヶ月）
- 支出内訳ドーナツチャート（今月）
- 支払い方法別支出サマリー
- 科目別支出サマリー
- 月初棚卸しバナー（毎月1日に先月未棚卸しの場合表示）
- 収支明細カードリスト（編集・削除・科目フィルタ）
- 月ナビ（前後月の切替）
- 残高繰越設定モーダル（⚙ボタン）
- 月末棚卸しモーダル（帳簿残高参照 → 実際残高入力 → 差異表示 → 締め確定）

### 入力フォーム（form.html）
- 収支区分：収入 / 支出 / ATM振替
- 支払い方法：楽天ペイ・PayPay・メルカリ（銀行口座） / 現金
- 収入受取：銀行口座 / 現金
- 金額クイック入力ボタン（¥500〜¥30,000）
- リアルタイム残高プレビュー
- STEP2 確認画面・STEP3 完了画面
- **STEP3 完了画面に「今日の入力履歴」を表示し、🗑 ボタンで削除可能**
  - 削除後は `allTransactions` を API から再取得し、残高プレビュー・履歴を全再計算
- **削除確認モーダル**（スタイル完全実装、PC・スマホ対応）

### 月次サマリー（summary.html）
- 月選択（前後ナビ + ジャンプセレクト、24ヶ月分）
- KPI：収入 / 支出 / 当月収支 / 棚卸し状況
- 棚卸し結果カード（銀行・現金の帳簿/実績/差異）
- ボックス別支出（A/B/C箱）vs 予算比プログレスバー
- カテゴリ別詳細（各箱の全科目 実績 vs 予算）
- 月次収支推移グラフ（過去12ヶ月 棒グラフ）
- 支払い方法トレンドグラフ（過去6ヶ月 積み上げ棒）
- 残高推移グラフ（銀行・現金・合計 折れ線 12ヶ月）
- 棚卸し差異履歴リスト

### 予算管理（budget.html）
- 月選択（前後ナビ + ジャンプセレクト）
- A/B/C ボックスオーバービュー（予算消化率バッジ）
- A箱・B箱・C箱ごとのカテゴリ別予算入力フィールド
- 実績表示・消化率プログレスバー
- **一括保存**（全カテゴリを一度に保存）
- **前月実績コピー**（前月の支出実績を今月の予算として一括設定）
- コピー確認モーダル

## データモデル

### テーブル: `transactions`
| フィールド | 型 | 説明 |
|---|---|---|
| id | text | UUID |
| date | text | YYYY-MM-DD |
| type | text | income / expense / transfer |
| category | text | 勘定科目名 |
| amount | number | 金額 |
| payment_method | text | rakuten/paypay/mercari/cash/bank_in/transfer_to_cash |
| memo | text | 任意メモ |

### テーブル: `balance_settings`
| フィールド | 型 | 説明 |
|---|---|---|
| id | text | UUID |
| month | text | YYYY-MM |
| bank_balance | number | 月初銀行口座残高 |
| cash_balance | number | 月初現金残高 |

### テーブル: `monthly_closings`
| フィールド | 型 | 説明 |
|---|---|---|
| id | text | UUID |
| month | text | YYYY-MM |
| bank_system | number | システム計算銀行残高 |
| cash_system | number | システム計算現金残高 |
| bank_actual | number | 実際の銀行残高（棚卸し入力値） |
| cash_actual | number | 実際の現金残高（棚卸し入力値） |
| bank_diff | number | 銀行差異 (actual - system) |
| cash_diff | number | 現金差異 (actual - system) |
| note | text | 棚卸しメモ |
| is_closed | bool | 締め確定フラグ |

### テーブル: `budgets`
| フィールド | 型 | 説明 |
|---|---|---|
| id | text | UUID |
| month | text | YYYY-MM |
| box | text | A / B / C |
| category | text | 勘定科目名（空=ボックス合計予算） |
| amount | number | 予算金額 |
| note | text | 備考 |

## 勘定科目・ボックス分類

| ボックス | 説明 | 科目 |
|---|---|---|
| A箱 | 事業経費 | 通信費・消耗品費・旅費交通費・支払手数料・接待交際費・新聞図書費 |
| B箱 | 個人消費 | 個人サブスク通信費・飲食費・美容被服費・趣味娯楽費・個人交際費予備費 |
| C箱 | 固定費 | 家族生活費・租税公課・法定福利費 |

## 推奨次ステップ

1. **CSVエクスポート** — 月次データをダウンロードしてExcel分析
2. **サブスク自動登録** — 毎月固定費の自動追加機能
3. **予算アラート** — 消化率80%超えで通知カード表示
4. **カレンダービュー** — 日別支出をカレンダー表示
5. **カスタムドメイン連携** — Cloudflare DNS設定で独自URLに

## スマホのキャッシュクリア手順

アプリを更新してもスマホで古いバージョンが表示される場合は、以下の手順でキャッシュを消去してください。

---

### 📱 iPhone（Safari）の場合

**方法A ― ページをハード再読み込み（最も簡単）**
1. Safari でアプリのURLを開く
2. アドレスバーを長押し → **「再読み込み」** をタップ
3. それでも変わらない場合 → 下記「方法B」へ

**方法B ― Safari の全キャッシュ消去**
1. 「設定」アプリ → **Safari** → 「履歴とWebサイトデータを消去」
2. Safari を再起動 → URLを開く直す

**方法C ― ホーム画面アイコンから開いている場合**
1. 一度ホーム画面のアイコンを**削除**（長押し → 削除）
2. Safari でURLを開き直す
3. 再度「ホーム画面に追加」

---

### 🤖 Android（Chrome）の場合

**方法A ― ページ強制リロード**
1. Chrome でURLを開く
2. 右上「⋮」メニュー → **「再読み込み」** を長押し → 「ハードリロード」

**方法B ― キャッシュ消去**
1. Chrome右上「⋮」→ 設定 → プライバシー → 「閲覧履歴データの削除」
2. 「キャッシュされた画像とファイル」にチェック → 削除
3. URLを開き直す

---

### 🔗 URLが正しいか確認する

正しいURLのパターン：
```
https://（あなたのドメイン）/index.html       ← ダッシュボード
https://（あなたのドメイン）/form.html        ← 収支入力
https://（あなたのドメイン）/budget.html      ← 予算管理
https://（あなたのドメイン）/summary.html     ← 月次サマリー
```

> ⚠ URLの末尾に `?v=...` などが付いていても正常です（キャッシュ対策）
